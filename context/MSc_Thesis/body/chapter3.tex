\chapter{Methodology} \label{chap:ch3}

This chapter describes how the negotiation capabilities of open-weight large language models were evaluated and how the two inference-time techniques studied in this thesis (Self-Refine and Team Negotiation) were implemented. Section~\ref{sec:na-framework} introduces the NegotiationArena framework by \textcite{bianchi_how_2024} shared across all experiments, its three game scenarios and the metrics used to score them. Section~\ref{sec:models-infra} presents the models under evaluation and the infrastructure used to run them. The remaining sections describe the experimental design for each of the three contributions: benchmarking open-weight models (Section~\ref{sec:benchmarking}), self-refine (Section~\ref{sec:self-refine}) and team-based negotiation (Section~\ref{sec:team-negotiation}).

\section{NegotiationArena Framework} \label{sec:na-framework}

\textcite{bianchi_how_2024} introduced NegotiationArena, an open-source framework to evaluate and analyze the negotiation abilities of LLM agents. The original work focused on proprietary models, some of which have since been deprecated; we keep the games and protocol unchanged but replace the agents with the open-weight models identified in Section~\ref{sec:models-infra} and expand it implementing two inference-time techniques (Sections~\ref{sec:self-refine} and~\ref{sec:team-negotiation}).

\subsection{Games} \label{sec:games}
% Explicar bem os três cenários: Ultimatum Game; Trading Game; Price Negotiation (BuySell)
% Possíveis ações; o que significa ganhar em cada um deles?; 

In each scenario, two agents alternate turns for up to a fixed number of rounds, or until one side terminates the game. Before each move, the agent reasons privately step by step; this reasoning is never revealed to the opponent.

\paragraph{Resource Exchange Scenario (Trading game).}
In the trading game, each party starts with a set of resources and aims to maximise their total holdings. The parties exchange proposals until one is accepted or the maximum number of rounds is reached. The agent with the greater gain wins.

\paragraph{Multi-Turn Ultimatum Game.}
In this variation of the classic Ultimatum Game \cite{doi:10.1126/science.1082976}, one agent receives all of the game's resources but must propose a split in order to keep any of it. The second agent can either accept, counter-propose a new split, or reject outright, causing both agents to lose all resources. If no agreement is reached within the maximum number of rounds, both agents lose everything. The winner is the agent with the higher share of the agreed split.

\paragraph{Seller and Buyer Scenario.}
Incomplete-information game in which a seller and a buyer negotiate the price of an item. Each party holds a private valuation: the seller knows its production cost and the buyer its willingness to pay. On each turn, a player may issue a \textsc{Proposal}, \textsc{Accept} the opponent's offer, or \textsc{Reject} and end the game with no deal. An agent wins if the agreed price is closer to the opponent's reservation value than to its own.

\paragraph{}
Every game runs for at most eight turns, after which it terminates without agreement if none has been reached. The system prompts and tag specifications for each game are available in Appendix~\ref{ap2:na-prompts}. The default parameters used are summarized in Table~\ref{tab:game-setup}.

\subsection{Evaluation Metrics}

% Usamos Win Rate (percentagem de jogos ganhos) e Average Payoff como usaram no NegotiationArena
% Para além disso analisamos a completion rate - percentagem de jogos que alcançam um estado final (por vezes não alcançam por um erro de protocolo/formato). Esta métrica é especialmente importante para medir o impacto do nosso mecanismo de retry 
Negotiation performance is measured using the two metrics adopted in the NegotiationArena (Win Rate and Average Payoff) complemented by a Completion Rate.

\begin{itemize}
    \item \textbf{Win Rate:} Percentage of \textit{completed} games in which a party obtained a higher payoff than the other. As in NegotiationArena, ties are ignored.
    \item \textbf{Average Payoff:} Average number of resources of each agent after the trade. Its definition is game-specific: in Price Negotiation it is the surplus ($\text{price} - \text{cost}$ for the seller, $\text{value} - \text{price}$ for the buyer); in Trading it is the net change in resource value ($\text{final} - \text{initial}$), which may be negative; and in Ultimatum it is the number of dollars retained.
    \item \textbf{Completion Rate:} Proportion of games that reach a final state. Because smaller open-weight models frequently violated the response format, this metric is reported alongside the others, and is the primary metric for the retry ablation.
\end{itemize}

% é usada quantização de 8 bits (bitsandbytes) nos modelos Gemma, Qwen e Mistral Small; os modelos Ministral já vêm quantizados em FP8 e são usados nessa precisão. Os modelos Qwen correm com o thinking mode desativado.

Open-weight models vary considerably in scale, so our objective was to group them into comparable parameter tiers in order to measure performance differences across model sizes. We defined three tiers: approximately 4–9B, 12–14B, and 24–27B parameters. 

For each tier, we selected the most recent available models from three families: Qwen, Mistral, and Gemma. Maintaining a consistent version across all tiers was not possible for Qwen and Mistral, so we included models from both Qwen 2.5 and Qwen 3, and from Ministral and Mistral Small, respectively. We initially considered Llama models as well, but we noticed difficulties in adhering to the required output format; given the additional computational overhead this would entail, we excluded them from the benchmark. The selected models are listed in Table~\ref{tab:experiment-models}.

We applied $8$-bit quantization (\texttt{bitsandbytes}) to Gemma, Qwen and Mistral-Small models; the Ministral models are already distributed in native FP8 precision. Generation uses temperature $0.7$ and top-p $0.9$ sampling, with a maximum of $1024$ new tokens per generation call.

We implemented a common agent built on the HuggingFace \texttt{transformers} library, abstracting model loading, quantization, and chat-template formatting behind a single interface. This design kept the experiments agnostic to the underlying model family and execution environment, allowing the same code to run unchanged across all three platforms.

Given the computational demands of the experiments, runs were distributed across three environments: the Kaggle free tier (2$\times$ NVIDIA T4) for smaller models, the MIA server (NVIDIA L40S), and the Deucalion HPC cluster (NVIDIA A100) \footnote{\url{https://rnca.fccn.pt/en/deucalion/}}.
